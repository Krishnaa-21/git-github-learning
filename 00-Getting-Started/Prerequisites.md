## 💻 2. Basic Command Line Familiarity

<div align="center">
<img src="https://raw.githubusercontent.com/Tarikul-Islam-Anik/Animated-Fluent-Emojis/master/Emojis/Objects/Keyboard.png" width="80">
<h3>Terminal Essentials</h3>
</div>

<br>

<table>
<tr>
<td width="50%">

<h3>❌ You Don't Need To Be:</h3>

<ul>
<li>❌ A terminal wizard</li>
<li>❌ A Linux expert</li>
<li>❌ A shell scripting master</li>
<li>❌ Comfortable with complex commands</li>
</ul>

<br>

<h3>✅ You Just Need To Know:</h3>

5 simple commands to get started:

```bash
pwd     # Print Working Directory (where am I?)
ls      # List files (what's here?)
cd      # Change Directory (move around)
mkdir   # Make Directory (create folder)
touch   # Create file (make new file)
```

</td>
<td width="50%">

<h3>🎯 Why Terminal Matters</h3>

<br>

<strong>Professional Reality:</strong>

<pre>
+ 95% of developers use Git from terminal
+ Git was designed for command-line usage
+ Terminal is faster than GUI tools
+ Essential for DevOps & automation
</pre>

<br>

<strong>Don't Worry!</strong>

<pre>
You'll learn as you go
→ Practice makes perfect
→ Mistakes are learning opportunities
→ Everyone starts somewhere
</pre>

</td>
</tr>
</table>

<br>

<details>
<summary><strong>🎓 Quick Terminal Cheat Sheet (Click to expand)</strong></summary>

<br>

```bash
# Navigate filesystem
pwd                    # Show current directory
ls                     # List files
ls -la                 # List with details
cd foldername          # Enter folder
cd ..                  # Go up one level
cd ~                   # Go to home directory

# Create files and folders
mkdir myfolder         # Create directory
touch myfile.txt       # Create file
mkdir -p path/to/dir   # Create nested directories

# File operations
cp file.txt copy.txt   # Copy file
mv old.txt new.txt     # Rename/Move file
rm file.txt            # Delete file
rm -rf foldername      # Delete folder

# View contents
cat file.txt           # Show file contents
```

</details>

## 🧠 7. Learn Concepts Before Commands

<div align="center">
<img src="https://raw.githubusercontent.com/Tarikul-Islam-Anik/Animated-Fluent-Emojis/master/Emojis/Objects/Books.png" width="80">
<h3>Understanding Over Memorization</h3>
</div>

<br>

<table>
<tr>
<td width="50%">

<h3>❌ The Wrong Approach</h3>

<br>

```bash
# Memorizing commands blindly
git commit -m "message"
git push origin main
git pull
git merge
git rebase
# ... and 100 more commands
```

<br>

<strong>Problems:</strong>
<ul>
<li>🤯 Information overload</li>
<li>😵 Easy to forget</li>
<li>🚫 No understanding why</li>
<li>⚠️ Dangerous in real scenarios</li>
</ul>

</td>
<td width="50%">

<h3>✅ The Right Approach</h3>

<br>

<strong>Think In Steps:</strong>

<pre>
1️⃣ Understand the Problem
        ↓
2️⃣ Learn the Concept
        ↓
3️⃣ Discover the Command
        ↓
4️⃣ Practice & Master
</pre>

<br>

<strong>Example:</strong>

<p>❓ <strong>Problem:</strong> "I want to save the current state of my project"</p>
<p>💡 <strong>Concept:</strong> Git creates snapshots (commits) of your work</p>
<p>⌨️ <strong>Command:</strong> <code>git commit -m "description"</code></p>

</td>
</tr>
</table>

<br>

<h3>🎯 Think In Workflows, Not Commands</h3>

<br>

<pre>
+ "I need to save my progress"            → git commit
+ "I want to try a new feature safely"    → git branch
+ "I need to combine my changes"          → git merge
+ "I want to share my code"               → git push
+ "I want to get others' updates"         → git pull
</pre>
