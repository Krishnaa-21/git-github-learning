# Git Push and Pull

## Introduction

In collaborative development, code exists both on the local machine and in a remote repository such as GitHub.

Git provides commands to synchronize changes between these environments.

Two essential commands for this workflow are:

- `git push`
- `git pull`

  

These commands are fundamental for sharing work, updating local projects, and collaborating with teams.

## Git Push

The `git push` command uploads local commits from your machine to a remote repository.

It is commonly used after completing local development work and committing changes.

In simple terms:

> Push sends your local changes to GitHub.

### Syntax

```bash
git push <remote-name> <branch-name>
```
Example:
```bash
git push origin main
```
