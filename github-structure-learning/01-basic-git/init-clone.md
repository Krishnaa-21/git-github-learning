## 📖 Overview

Before working with Git and GitHub projects, two foundational commands are essential for every developer:

<div align="center">

| Command | Purpose | Use Case |
|---------|---------|----------|
| `git init` | Initialize new repository | Starting a new project |
| `git clone` | Copy existing repository | Working on existing project |

</div>

These commands help you **start a new Git project** or **copy an existing one** to your local machine.

> 💡 **Understanding these commands clearly is critical for every developer.**

---

## 🆕 1. Git Init

<div align="center">

<img src="https://img.shields.io/badge/Command-git_init-orange?style=for-the-badge&logo=git" />

</div>

### 📝 Definition

`git init` **initializes a new Git repository** in your current project folder.

It tells Git:

> ✨ **"Start tracking this project."**

After running this command, Git creates a hidden `.git` directory that stores:

<table>
<tr>
<td width="50%">

**Repository Data:**
- 📜 Commit history
- 🌿 Branch information
- ⚙️ Configuration files

</td>
<td width="50%">

**Tracking Information:**
- 📋 Staging area
- 🏷️ Repository metadata
- 🔖 References and tags

</td>
</tr>
</table>

---

### 📋 Syntax

```bash
git init


git init <project-name>
```
